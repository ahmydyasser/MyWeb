Install Windows beside linux

( Fixing the "No Device Drivers Were Found" )

It sounds like you want to install Windows on your second disk (/dev/nvme0n1), which currently has Arch Linux on it, and you're running into the “No Device Drivers Were Found” error during installation. Let’s walk through the steps to prepare that disk for Windows and address the driver issue.

Steps to Prepare /dev/nvme0n1 for Windows Installation , BTW /dev/nvmme1n1 is Kali "don't touch it"

1. Backup Any Important Data

If there’s any data on the /dev/nvme0n1 disk that you want to keep, make sure to back it up before proceeding, as these steps will erase all existing data on that disk.

2. Delete Existing Partitions

You’ll need to delete the existing partitions on /dev/nvme0n1 to make it available for Windows. You can do this using fdisk or parted. Here, I’ll guide you using fdisk:

  1. Open Terminal and Start fdisk:

    sudo fdisk /dev/nvme0n1

  2. List Partitions:

    • Press p to print the current partitions. This will show you what’s on the disk.

  3. Delete Existing Partitions:

    • Press d and then enter the partition number (e.g., if there’s one partition, just press d and hit Enter). Repeat this until all partitions are deleted.

  4. Write Changes:

    • Press w to write the changes and exit fdisk.

3. Format the Disk (Optional)

You can skip this step since Windows will handle formatting, but if you want to do it manually, you can create a new NTFS partition using parted:

  1. Start parted:

    sudo parted /dev/nvme0n1

  2. Create a New Partition Table:

    (parted) mklabel gpt

  3. Create a New Partition:

    (parted) mkpart primary ntfs 0% 100%

  4. Exit parted:

    (parted) quit

4. Install Windows

Now that you’ve prepared the disk, follow these steps to install Windows:

  1. Use a Bootable Windows USB: Make sure your USB drive is bootable with Windows 11. If you created it on macOS, it should be set up correctly. If not, consider recreating it using a Windows PC with the Media Creation Tool, as mentioned earlier.

  2. Boot from USB:

    • Restart your computer and access the boot menu (usually by pressing F9 or Esc during startup).
    • Select the USB drive to boot from.

  3. Start the Installation:

    • When you reach the installation screen, choose Custom Install.

  4. Select the Disk:

    • You should see the unallocated space on /dev/nvme0n1. Select that space for installation.
    • If you formatted the disk as NTFS in the previous step, it should show up as a partition; otherwise, you may see unallocated space.

5. Address Driver Issues

If you still encounter the “No Device Drivers Were Found” error:

  1. Ensure UEFI Mode:

    • Make sure you’re booting the USB in UEFI mode (not Legacy). Check your BIOS settings.

  2. Disable Secure Boot:

    • Temporarily disable Secure Boot in BIOS.

  3. Check USB Drive:

    • Sometimes, the USB drive itself may have issues. Try using a different USB drive or port.

  4. Load Drivers Manually (if necessary):

    • If you have access to a driver package, you can load drivers from another USB drive during installation if prompted.

Conclusion

After completing these steps, Windows should install without the driver issues. Let me know if you encounter any problems along the way!

It looks like you’ve successfully accessed the fdisk utility for /dev/nvme0n1. Since it’s currently using a DOS partition table and you want to remove the existing partitions, here are the steps to delete the partitions and create a new one for Windows:

Steps to Delete Existing Partitions

  1. List Existing Partitions:

    • Since the p command didn't show any partitions, it’s possible that there are no visible partitions or they are not being displayed correctly. In this case, proceed with the next steps to delete any partitions you might be aware of.

  2. Delete Existing Partitions:

    • To delete a partition, use the d command. If you’re not sure how many partitions are present, you can try issuing the d command directly. It will prompt you to specify the partition number.
    • If there are no partitions, you can skip to creating a new partition.

  3. Create a New Partition Table (Optional)

    • Since you want to install Windows, it’s a good idea to create a new partition table.
    • Type the following command to create a new partition table:
      Command (m for help): o
    • This will create a new empty DOS partition table.

  4. Create a New Partition:

    • Now, create a new partition by typing:
      Command (m for help): n
    • You’ll be prompted to choose the partition number (just hit Enter for default), and then specify the first and last sectors (you can just hit Enter to use the default values to use the entire disk).

  5. Change the Partition Type:

    • If you want to set the partition type to NTFS (which Windows requires), you can type:
      Command (m for help): t
    • Enter the partition number you just created, and then type 7 to set the partition type to NTFS.

  6. Write Changes to Disk:

    • Finally, write the changes to the disk by typing:
      Command (m for help): w
    • This will apply the changes you made. After this, the disk is ready for Windows installation.

Next Steps

After preparing the partition:

  1. Boot from the Windows Installation USB:

    • Restart your computer and boot from the USB drive as described earlier.

  2. Select the Newly Created Partition:

    • During the Windows installation, select the partition you created for installation. Windows will format it to NTFS if it’s not already formatted.

  3. Complete the Installation:

    • Follow the on-screen instructions to complete the Windows installation.

Troubleshooting the “No Device Drivers Were Found” Error

If you encounter the error again:

  • Ensure that the USB drive is correctly formatted and created.
  • Try using different USB ports, preferably USB 2.0 if available.
  • Check your BIOS settings for UEFI and Secure Boot options.

Since there are no partitions to delete, let’s create a new partition directly. Here’s how you can proceed:

Steps to Create a New Partition

  1. Create a New Partition:

    • Start by creating a new partition by typing:
      n
    • You'll be prompted to specify the partition number. Just hit Enter to accept the default (usually 1).

  2. Specify the First Sector:

    • Next, you’ll be prompted for the first sector. Just hit Enter to use the default, which should start from the beginning of the disk.

  3. Specify the Last Sector:

    • When prompted for the last sector, just hit Enter again to allocate the entire disk space for the partition.

  4. Set the Partition Type:

    • To set the partition type to NTFS, type:
      t
    • When prompted for the partition number, enter 1 (or the partition number you just created).
    • Then type 7 to set the partition type to NTFS.

  5. Write Changes to the Disk:

    • Finally, write the changes to the disk by typing:
      w
    • This will create the new partition and apply the changes.

Check Your Changes

After you've completed these steps, you can verify that the partition has been created successfully:

  1. Re-open fdisk:

    sudo fdisk /dev/nvme0n1

  2. Print the Partition Table:

    • Type:
      p
    • You should now see your newly created partition listed.

Proceed with Windows Installation

Now that you have created the partition, you can boot from the Windows installation USB and select this partition during the installation process.

  • Follow the same steps as before, making sure to choose the correct partition for Windows installation.

If You Encounter the “No Device Drivers Were Found” Error Again

  • Make Sure the USB Drive is Properly Created: If the error persists, ensure the USB is properly created and recognized by the BIOS.
  • Use Different USB Ports: Use USB 2.0 ports if available, as they sometimes work better for booting.

It looks like you've successfully created a new partition on /dev/nvme0n1, but it appears to have a vfat (FAT32) signature instead of the intended NTFS format. Here’s how to proceed from here to ensure you set up the partition correctly for Windows installation:

Remove the vfat Signature

Since the partition was created with a vfat signature, you’ll want to remove it. When prompted with:

Do you want to remove the signature? [Y]es/[N]o:
  • Type Y and press Enter. This will remove the existing FAT32 signature.

Change the Partition Type to NTFS

  1. Change the Partition Type: After you remove the signature, you can change the partition type to NTFS:

    • Enter the t command:
      t
    • When prompted for the partition number, enter 1.
    • Then type 7 to set the type to NTFS:
      7

  2. Write the Changes: Finally, you’ll need to write the changes to the disk:

    w

Format the Partition as NTFS

Now that the partition is created and set as NTFS, you should format it:

  1. Format the Partition: You can format the partition to NTFS directly from your Kali Linux environment using the mkfs.ntfs command. Run:
    sudo mkfs.ntfs -f /dev/nvme0n1p1

Verify the Changes

After formatting, verify that the partition is correctly formatted:

  1. Re-open fdisk:

    sudo fdisk /dev/nvme0n1

  2. Print the Partition Table:

    • Type:
      p
    • You should now see your partition as NTFS.

Proceed with Windows Installation

Now, you can reboot and boot from your Windows installation USB:

  1. Select the Partition for Windows Installation: During the installation process, select the NTFS partition you just created.

Troubleshooting the “No Device Drivers Were Found” Error

If you encounter the same error again during Windows installation:

  • Use USB 2.0 Ports: If available, use USB 2.0 ports for the installation USB.

Great! It looks like you've successfully created and formatted the partition on your second drive as NTFS. Now, you're all set to proceed with the Windows installation. Here are the next steps:

1. Reboot Your Computer

  • Make sure your bootable Windows 11 USB drive is connected to the computer.

2. Enter BIOS/UEFI Settings

  • When you power on your PC, press the key to enter BIOS/UEFI settings (usually F2, F10, F12, or Esc depending on your computer model).

3. Set Boot Order

  • In the BIOS/UEFI menu, navigate to the boot options and ensure that the USB drive is set as the first boot device.
  • Save the changes and exit BIOS.

4. Start Windows Installation

  • Your computer should now boot from the USB drive.
  • Follow the prompts to start the Windows installation process.

5. Select Installation Type

  • When you reach the installation type screen, choose Custom: Install Windows only (advanced).

6. Select the NTFS Partition

  • You should see the partition you created on /dev/nvme0n1 listed (it will show the size and indicate that it's NTFS).
  • Select this partition for the Windows installation.

7. Continue with the Installation

  • Follow the remaining prompts to complete the installation.

Troubleshooting “No Device Drivers Were Found”

If you still encounter the “No device drivers were found” error:

  1. Try a Different USB Port: If you're using a USB 3.0 port, switch to a USB 2.0 port.
  2. Check USB Format: Make sure the USB drive is formatted correctly as FAT32 or NTFS (it should be FAT32 for the installation media created with the Media Creation Tool).
  3. Disable Secure Boot: If it's not already done, disable Secure Boot in the BIOS settings.

Once the installation is complete, you'll be able to boot into Windows from the newly created NTFS partition. Let me know if you need any further help or if something doesn't go as planned!
